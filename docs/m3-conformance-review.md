# M3 Conformance Review — `@ngguide/ui` vs `docs/`

> **Scope:** every component implementation under `libs/ui/` audited against its transcribed Material Design 3 spec doc under [`docs/components/`](components). The kit is built from scratch to M3 strictly (not a wrapper around Angular Material), so any deviation from the doc is a finding.
>
> **Method:** 28 components × (1 auditor + 1 independent skeptic) = 56 agents. Each auditor read the M3 doc + the full implementation (`.ts` + `.css` + the shared tokens in `libs/ui/src/styles/tokens/`) and listed every concrete deviation grounded in a `file:line`. An independent verifier then re-checked each claim against the real files, rejecting false positives and adding clear misses. Only **verified** discrepancies are recorded below.
>
> **Captured 2026-06-04** via the `m3-conformance-review` workflow.

## Headline

- **232 confirmed discrepancies** across 28 components — 🔴 42 high · 🟠 64 medium · 🟡 126 low.
- **16 claims rejected** during adversarial verification (auditor misreads — not recorded).
- **Fix target:** 172 point at the implementation, 15 at the doc (transcription artifacts), 45 either/unclear.

Severity is conformance impact, not a bug ranking: 🔴 high = a spec'd variant/size/state/measurement is absent or wrong; 🟠 medium = a concrete value or behaviour deviates; 🟡 low = a minor token/aria/edge gap.
